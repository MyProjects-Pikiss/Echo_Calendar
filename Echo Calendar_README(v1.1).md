# Echo Calendar

## v1.1 (Offline-first, Online-optional 명세)

---

## 1. 프로젝트 개요

Echo Calendar는 **개인 기록을 안정적으로 남기고, 다시 찾아보기 위한 캘린더 앱**입니다.

---

## 2. 동작 모드 정의 (중요)

### 2.1 오프라인 기본 모드 (필수)

Echo Calendar의 기본 동작 모드이며, 네트워크 연결 없이 완전히 사용 가능합니다.

제공 기능

* 월간 캘린더 기반 이벤트 조회
* 날짜 선택 → 해당 날짜의 이벤트 목록 확인
* 텍스트 기반 이벤트 추가 / 수정 / 삭제 (CRUD)
* Full-Text Search(FTS)를 이용한 로컬 검색

모든 데이터는 로컬 Room DB가 **정본(Source of Truth)**이며,
오프라인 상태에서도 기록과 조회에 제약이 없습니다.

### 2.2 온라인 부가 모드 (선택)

네트워크 연결 시에만 활성화되는 **AI 기반 입력·해석 모드**입니다.
이 모드는 기존 오프라인 기능을 대체하지 않으며, 기록 입력을 보조하는 역할만 수행합니다.

제공 기능

* 음성 입력(STT) 및 자연어 텍스트 입력
* AI(서버 + LLM)를 통한 입력 내용 해석
* 해석 결과를 기반으로 한 **이벤트 CRUD 제안 생성**

동작 방식

* AI는 음성/자연어 입력을 해석하여 이벤트 생성·수정·삭제에 대한 **제안 데이터**를 생성
* AI는 로컬 DB에 직접 접근하지 않음
* 제안된 변경 사항은 반드시 사용자 확인 팝업을 거친 후에만 실제 DB에 반영됨

운영 원칙

* AI 기능은 서버 및 LLM과의 통신이 필요하므로 **온라인 전용 모드**로 동작
* 오프라인 상태에서는 관련 UI가 비활성화됨
* 온라인 기능 실패 시에도 기존 로컬 데이터에는 영향 없음

## 3. 핵심 기능 범위 (v1.1 기준)

### 3.1 캘린더 & 기록

* 월간 캘린더 화면(MonthCalendarScreen)
* 날짜 선택 시 해당 날짜의 이벤트 목록 표시
* 빠른 텍스트 입력을 통한 이벤트 기록
* 카테고리 기반 이벤트 분류

### 3.2 검색

* 로컬 DB 기반 Full-Text Search
* 검색 결과 선택 시 캘린더의 해당 날짜로 이동
* 검색 기능은 항상 오프라인에서 사용 가능

### 3.3 알림

* 이벤트 기반 알림(EventAlarm)
* 로컬 스케줄링 기반 알림 처리

---

## 4. 데이터 변경 원칙 (중요)

Echo Calendar에서는 **데이터 변경의 주체가 사용자이든 AI이든 동일한 규칙을 적용**합니다.

* 이벤트 생성 / 수정 / 삭제(CRUD)는 항상 사용자 확인을 거쳐야 함
* AI(음성/자연어 해석 포함)는 직접 DB를 갱신하지 않음
* AI는 "변경 제안"만 생성하며, 실제 반영 여부는 사용자가 확인 팝업을 통해 결정

이를 통해 다음을 보장합니다.

* 오기록 및 오해석으로 인한 데이터 손상 방지
* 모든 기록 변경에 대한 사용자 통제권 유지

## 5. 기술 스택 및 사용 목적

### Android

* **Kotlin**

  * 앱 전체 로직 구현
  * 안정적인 타입 기반 데이터 처리

* **Jetpack Compose**

  * 캘린더, 검색, 이벤트 리스트 UI 구성
  * 상태 기반 UI 렌더링

* **Flow / StateFlow**

  * 캘린더, 검색, 이벤트 상태 관리
  * 화면 간 상태 동기화

### 로컬 데이터 (Room Database)

* **Room Database**

  * 앱의 모든 기록 데이터 저장소
  * 오프라인 정본 데이터 관리

* **Category / Event / Label / EventLabel**

  * 이벤트 구조화 및 분류

* **EventFts (FTS4)**

  * 오프라인 텍스트 검색 지원

* **EventAlarm**

  * 이벤트 기반 로컬 알림 관리

### 온라인 (계획)

* **Django + DRF**

  * 자연어 기록 해석 서버
  * 클라이언트와의 API 계약 제공

* **LLM API**

  * 음성/텍스트 입력을 구조화된 이벤트 데이터로 변환
  * 직접 DB 접근 없이 제안 데이터만 생성
